# UI Polish Pass: Priorities 1-5

**Date:** 2026-02-09
**Status:** Planned — not yet implemented
**Scope:** 5 changes to upgrade VoxFlow's menu bar UI from prototype to premium

---

## Context

VoxFlow's UI is functional but generic — system fonts at 6+ hardcoded sizes, opaque gray backgrounds, zero animations, and `.menuBarExtraStyle(.menu)` which renders through NSMenu and strips custom SwiftUI styling. This pass upgrades the UI from prototype to premium with 5 targeted changes.

**Decision made:** User approved `.window` style switch. System menu items (Setup, Dashboard, Settings, Quit) will be rebuilt as a compact footer inside `CommandPaletteView`.

---

## Priority 1: Switch MenuBarExtra to `.window` style

**File:** `Sources/VoxFlowApp/VoxFlowLocalApp.swift`

Replace `.menuBarExtraStyle(.menu)` with `.menuBarExtraStyle(.window)`. This renders `CommandPaletteView` as a full SwiftUI popover instead of through NSMenu, unlocking all other visual improvements.

- Remove system menu items (Setup, Dashboard, Settings, Quit) from the MenuBarExtra block — they don't render in `.window` style
- Add `onOpenSetup` closure parameter to `CommandPaletteView`

**File:** `Sources/VoxFlowApp/Views/CommandPaletteView.swift`

Add a compact footer row at the bottom of the body with icon buttons for Setup, Dashboard, Settings (via SettingsLink), and Quit.

---

## Priority 2: Replace opaque backgrounds with materials

Replace all `Color.gray.opacity(0.XX)` backgrounds with SwiftUI materials:

| Current | Replacement | Usage |
|---------|-------------|-------|
| `Color(nsColor: .windowBackgroundColor)` | Remove — `.window` provides its own | Main background |
| `Color.gray.opacity(0.10)` | `.ultraThinMaterial` | Text preview boxes |
| `Color.gray.opacity(0.08)` | `.quaternary` | Card backgrounds |
| `Color.gray.opacity(0.06)` | `.quaternary.opacity(0.5)` | Recent row backgrounds |
| `Color.gray.opacity(0.14)` | `.regularMaterial` | Unselected chip backgrounds |

For tinted containers (privacy, translation, meeting), layer tinted color over material:
```swift
.background {
    RoundedRectangle(cornerRadius: 10)
        .fill(.ultraThinMaterial)
        .overlay(Color.green.opacity(0.08))
}
```

**Files:** `CommandPaletteView.swift`, `DashboardPanelView.swift`, `ModeChip.swift`

---

## Priority 3: Add panel transition animations

- **Panel switching:** `.animation(.smooth(duration: 0.25), value: activePanel)` with `.transition(.opacity.combined(with: .move(edge: .trailing)))`
- **Session state transitions:** `.animation(.smooth(duration: 0.3), value: state.sessionState)`
- **Error appearance:** `.transition(.opacity.combined(with: .move(edge: .top)))`
- **Recording badge pulse:** `.scaleEffect` with `.easeInOut(duration: 1.2).repeatForever(autoreverses: true)`

**File:** `CommandPaletteView.swift`

---

## Priority 4: Recording state overhaul

When `state.sessionState == .recording`, replace the entire `mainDictationCard` content with a focused recording view showing only:
- Animated 5-bar waveform (red, varying heights, spring animations)
- Large monospaced duration timer (e.g., "2.4")
- "Release hotkey to transcribe" caption

When `state.sessionState == .transcribing`, show a centered `ProgressView` + "Transcribing..." label.

All other UI (panels, cleanup modes, buttons) hidden during recording/transcribing states.

**File:** `CommandPaletteView.swift`

---

## Priority 5: Tighten type scale to 4 roles

**New file:** `Sources/VoxFlowApp/Views/VFDesignTokens.swift`

```swift
enum VF {
    static let titleFont = Font.system(size: 15, weight: .semibold)
    static let bodyFont = Font.system(size: 13)
    static let labelFont = Font.system(size: 12, weight: .medium)
    static let captionFont = Font.system(size: 11)

    static let cornerSmall: CGFloat = 8
    static let cornerMedium: CGFloat = 10
    static let cornerLarge: CGFloat = 12

    static let spacingSmall: CGFloat = 8
    static let spacingMedium: CGFloat = 12
    static let spacingLarge: CGFloat = 16
}
```

Replace all hardcoded `.system(size: N, weight: W)` calls in `CommandPaletteView`, `DashboardPanelView`, and `ModeChip` with `VF.*` tokens. Consolidate 6+ font sizes down to 4 semantic roles.

**Do NOT touch:** `DashboardWindowView`, `SettingsView`, `SetupWizardView` — separate windows, different visual context.
